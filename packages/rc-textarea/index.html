Themes

All visual styling flows through CSS custom properties on the host element. Decoration colour palettes are injected per-editor via setDecorationStyles() — no shadow-piercing selectors needed.

Syntax Highlighting — Plugin API

usePlugin() hands mirror rendering to any external parser or highlighting library. Here highlight.js runs on every keystroke via the plugin's highlight() callback. Its theme stylesheet is injected into the shadow root with adoptStyleSheet() — necessary because light-DOM stylesheets don't pierce the shadow boundary. Try editing the code.

Diagnostics — All Severities

Diagnostics render Error Lens–style, inline at the end of their line. createIcon() returns any HTMLElement to prepend as a leading icon. Wavy underlines on the error and warning tokens use box-shadow on mark decorations — never padding or margin, which would shift character positions in the mirror relative to the textarea.

Inline Widgets

Widgets occupy exactly 1ch — matching a U+2007 FIGURE SPACE character embedded in the textarea value at each widget position. This keeps selection highlights and cursor positions aligned between the textarea and the mirror. widgetPlacement: 'before' places the widget before the remaining text in the range; 'after' places it after. Figure-spaces are stripped from host.value, clipboard copy, paste, and form submission so consumers never see them.

Pattern Matching

addPattern() registers a regex that re-matches on every keystroke — no manual update needed. Multiple patterns can be active simultaneously. Try editing the text below.

Auto-Grow

autoGrow makes the editor expand to fit content — the mirror drives the host height, so no explicit height is needed.
