rc-textarea

Line numbers · word wrap · mark & line decorations · diagnostics · pattern matching · CSS custom property theming.

Syntax Highlighting — Plugin API

usePlugin() hands mirror rendering to any external parser or highlighting library. Here highlight.js runs on every keystroke via the plugin's highlight() callback. Its theme stylesheet is injected into the shadow root with adoptStyleSheet() — necessary because light-DOM stylesheets don't pierce the shadow boundary. Both the hljs CSS and the editor chrome (--rc-textarea-* custom properties) swap together on dark-mode toggle. Try editing the code. Theme: GitHub Light/Dark · font: ui-monospace / Cascadia Code, 13 px.

Diagnostics — All Severities

Diagnostics render Error Lens–style, inline at the end of their line. createIcon() returns any HTMLElement to prepend as a leading icon. Wavy underlines use box-shadow on mark decorations — never padding or margin, which would shift character positions in the mirror relative to the textarea. Theme: One Dark (fixed) · font: Courier New, 14 px / 1.65.

Pattern Matching

addPattern() registers a regex that re-matches on every keystroke — no manual update needed. Multiple patterns can be active simultaneously. Colour classes reference page-level CSS custom properties so they adapt automatically when dark mode is toggled. Try editing the text. Theme: auto (follows page) · font: system mono, 12 px / 1.45.

Auto-Grow

autoGrow makes the editor expand to fit its content — the mirror drives the host height, so no explicit height is needed. Theme: auto (follows page).
